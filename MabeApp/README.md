# Nexus HR — macOS HR Analytics Dashboard

A production-ready SwiftUI macOS dashboard for HR analytics, built with MVVM + Combine, Apple HIG compliance, and Swift Charts.

## Requirements

- macOS 14 Sonoma or later
- Xcode 15.0+
- Swift 5.9+

## Project Setup in Xcode

1. Create a new **macOS App** project in Xcode
2. Set **Bundle Identifier** to `com.yourcompany.nexushr`
3. Set **Minimum Deployment Target** to `macOS 14.0`
4. In **Signing & Capabilities**, enable **App Sandbox** (required for production)
5. Delete the default `ContentView.swift` and `AppName.swift` generated by Xcode
6. Drag all folders from this package into your Xcode project navigator
7. Ensure **"Copy items if needed"** and **"Add to targets: NexusHR"** are checked
8. Add the **Charts** framework: Project → Target → Frameworks → `+` → Charts.framework

## Integration Points

All locations where you connect real ML models or APIs are marked with:
```swift
// INTEGRATION POINT: ...
```

### Connecting Real Data (DashboardService.swift)
Replace `MockDashboardService` with `LiveDashboardService` and configure:
```swift
// In NexusHRApp.swift:
@StateObject private var dashboardVM = DashboardViewModel(service: LiveDashboardService())
```

Set your API token:
```bash
NEXUSHR_API_TOKEN=your_token_here xcodebuild ...
```

### Connecting CoreML Models (MLOrchestrator.swift)
```swift
private var classifier: NLModel?  // Your CreateML intent classifier
private var sentimentModel: NLModel?  // Your sentiment NLModel

func classify(query: String) async throws -> QueryCategory {
    let tagger = NLTagger(tagSchemes: [.tokenType])
    // ... your CoreML inference here
}
```

### Real-time Updates (DashboardViewModel.swift)
Replace `Timer.publish` with a WebSocket or SSE publisher:
```swift
private func setupAutoRefresh() {
    webSocketPublisher
        .receive(on: RunLoop.main)
        .sink { [weak self] _ in Task { await self?.loadAll() } }
        .store(in: &cancellables)
}
```

## Architecture

- **Pattern**: MVVM + Combine
- **Async**: `async/await` with `@MainActor` on all ViewModels
- **Charts**: Swift Charts (`import Charts`) — no third-party dependencies
- **Privacy**: `DataAnonymizer` runs SHA-256 hashing + regex PII stripping on all records before display
- **Dark Mode**: `@Environment(\.colorScheme)` wired in every View; color constants ready to expand

## Accessibility

- All charts have Spanish `.accessibilityLabel()` + `.accessibilityValue()` 
- KPI cards VoiceOver: "Tickets evitados: 1,284, incremento del 12% esta semana"
- macOS `Table` provides native keyboard navigation (arrow keys, Tab)
- Color is never used as the sole differentiator — icons/shapes always accompany color
- Contrast ratio ≥ 4.5:1 enforced by color constants (`textPrimary #1A1A1A` on `background #FDFFFF`)

## Development vs Production

| Service | Usage |
|---|---|
| `MockDashboardService` | Previews, UI tests, development |
| `LiveDashboardService` | Production — configure `baseURL` + auth |

The mock service adds a configurable simulated latency (default 0.3s) to surface loading states during development.
